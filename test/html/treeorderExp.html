树排序

树排序组件treeorder是对树组件tree功能的扩展。PR #1420

基本原理

tree组件中的数据源没有对外提供,在treeorder中缓存用户传入的数据源,然后根据需要(比如排序)进行处理,通过修改传入数据源信息达到排序等效果

排序说明

  • 默认是子节点越多的节点越靠上
  • 在实例的search方法中传入搜索内容是采用内容排序:即匹配到的内容越多的节点越靠上,匹配到的节点强制展开,未匹配到的节点收起
  • 排序的具体效果遇到特殊数据源可能有所差异
  • 可以在原始tree组件中自行修改传入的数据源达到自定义排序规则的效果

方法 - 树渲染

layui.tree(option, deploy)可以创建并返回一个树实例对象

请注意:此实例对象在treeorder中会修改它原来的方法,所以与从tree中获取的实例对象是有区别的

  • 参数option必填,除了树的配置项还新增了一个配置项
  • 参数deploy用来标记是否使用treeorder来管理树的数据源,默认是false;使用treeorder来渲染树就要填true。

在tree的配置参数中新增一个参数,其使用方法可以参考接下来的数据源介绍

属性名 描述 类型 默认值
customName.parentid 自定义parent节点名称,节点父节点的id(顶层节点为0) string parentid

数据源

  • tree可以接收一个对象型的数据源(即数据源是封装成一个对象,子节点以父节点下面的children来表示)

  • treeorder除了接收上面对象型的数据源,还可以接收列表型数据源

  • 列表型数据源:数据以list的方式传入,无论父节点还是子节点都是list中的对象,父子节点之间以parentid关联。子节点的parentid指向父节点的id,顶层节点的parentid设置成0

  • 上面的idparentid都可以在配置项customName中自定义

实例方法

tree.render会返回树的实例treeInst,通过增加和修改将这个实例的方法来实现各种效果

再次强调:此实例对象在treeorder中会修改它原来的方法,所以与从tree中获取的实例对象是有区别的

重载(修改)

treeInst.reload(option)可以根据传入的配置项来重载树

  • 参数option必填,树的配置项

搜索(新增)

treeInst.search(searchContent, values)可以根据传入的信息在树里面搜索,具体排序规则可以参考上面的排序规则

  • 参数searchContent可选,需要匹配搜索的信息,匹配数据源中customName.title对应的信息
  • 参数values可选,类型string/array,设置当前选中的节点,内容是节点的id,对应id的节点checked属性会设置为true,其它的会设置成false
